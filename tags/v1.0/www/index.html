NCron – .NET scheduling service

NCron is a task scheduling service, written completely in managed C#. The service will execute tasks written in any .NET-language, in a (almost) any possible time pattern.

History

The NCron project is a spin-off from the Whistleblower project. As a system monitoring framework, Whistleblower needs to do a lot of task scheduling, and when the scheduling logic started to grow, it seemed like a good idea to split up the project.

As of now, both NCron and Whistleblower is maintained by one lone developer, Jørn Schou-Rode of Denmark. Both projects are used by some of Jørn's clients, but the further development is mainly done in his spare time. If you have anything to say or add to the projects, do not hestitate to drop me an e-mail.

Usage / distributions

NCron is packed and distributed for two distinct types of usage:

Utility library
The core functionality of NCron is all placed within the NCron.dll assembly, which can be referenced by third-party .NET-projects as they need to schedule some task(s). Amongst a set of model and helper classes, lives the SchedulingService class, which can easily be configured and installed – possibly “disguised” under the name of the third-party project.
Stand-alone service
If you just need a service which will run some of your method(s) in some given time pattern(s), the stand-alone package might be the easiest way to go. Simply install the NCron scheduling service, modify its configuration files, and off you go!
